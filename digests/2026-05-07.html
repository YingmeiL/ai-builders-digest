Vercel 开源 deepsec
Guillermo Rauch (Vercel CEO)
安全

Vercel 开源 AI Agent 驱动的深度安全扫描框架。 (1,229 赞)

Rauch 称这是为内部使用构建的工具，经过实战验证后开源:

  • 全本地运行: 在个人设备上执行，用你已有的 AI 订阅，代码不上传云端
  • AI Agent 驱动: 比传统静态扫描更智能，能发现「难以察觉的问题」
  • 工作流: 正则初筛 → 定向文件分析 → 二次验证 (降低误报) → 贡献者识别 → 输出到项目管理工具
  • 可扩展: 支持远程 Vercel Sandbox 千级并发 + 模块化插件框架 (可自建检测规则)
背景

Vercel 在 4 月经历了安全事件 (第三方 AI 服务被入侵导致员工凭证泄露)。deepsec 的开源是 Vercel 从「被攻击者」到「安全工具提供者」的战略转变。这也呼应了 Anthropic Project Glasswing 的理念: 用 AI 加速防御。

行动建议: 如果你的团队有大型代码库，值得试用 deepsec 做一次全面安全扫描。全本地运行意味着不用担心代码泄露。

Claude Code Auto Mode -- 更安全地跳过权限确认

这篇博客揭示了一个有趣的工程问题和解决方案:

问题: Claude Code 默认每次操作都要用户确认 → 导致「审批疲劳」→ 用户 93% 的时间都在盲目点「同意」→ 反而更不安全。

现有方案的局限:

  • 沙箱隔离: 安全但高维护 -- 每个新能力都需要配置，需要网络/宿主访问的功能会被阻断
  • --dangerously-skip-permissions: 零维护但完全不安全

Auto Mode 方案: 在完全自主和完全手动之间找到平衡点

  • 安全分类器自动判断操作风险等级
  • 低风险操作自动执行，高风险仍需确认
  • 安全性随分类器覆盖率和模型判断力提升而持续改进
真实事故案例 (为何需要 Auto Mode):
  • Agent 因误解指令删除了远程 git 分支
  • Agent 将工程师的 GitHub auth token 上传到内部计算集群
  • Agent 尝试对生产数据库执行迁移
对开发者的意义: 如果你在日常工作中频繁使用 Claude Code，Auto Mode 可以大幅减少操作摩擦，同时保持安全底线。这是「Agent 自主性」和「安全性」之间工程权衡的优秀案例。
